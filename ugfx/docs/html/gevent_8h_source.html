45 // This object can be typecast to any GEventXxxxx type to allow any sub-system (or the application) to create events.
46 // The prerequisite is that the new status structure type starts with a field named 'type' of type 'GEventType'.
47 // The total status structure also must not exceed GEVENT_MAXIMUM_SIZE bytes.
48 // For example, this is used by GWIN button events, GINPUT data streams etc.
49 typedef union GEvent_u {
50  GEventType type; // The type of this event
51  char pad[GEVENT_MAXIMUM_SIZE]; // This is here to allow static initialisation of GEventObject's in the application.
52 } GEvent;

85 /* How to listen for events (act as a Listener)...
86  1. Get handles for all the event sources you are interested in.
87  2. Initialise a listener
88  3. Attach sources to your listener.
89  - Sources can be attached or detached from a listener at any time.
90  - A source can be attached to more than one listener.
91  4. Loop on getting listener events
92  5. When finished detach all sources from the listener
93 
94  How to create events (act as a Source)...
95  1. Provide a funtion to the application that returns a GSourceHandle (which can be a pointer to whatever the source wants)
96  2. Whenever a possible event occurs call geventGetSourceListener to get a pointer to a GSourceListener.
97  This will return NULL when there are no more listeners.
98  For each listener - check the flags to see if an event should be sent.
99  - use geventGetEvent() to get the event buffer supplied by the listener
100  and then call geventSendEvent to send the event.
101  - Note: geventGetEvent() may return FALSE to indicate the listener is currently not listening and
102  therefore no event should be sent. This situation enables the source to (optionally) flag
103  to the listener on its next wait that there have been missed events.
104  - Note: The GSourceListener pointer (and the GEvent buffer) are only valid between
105  the geventGetSourceListener call and either the geventSendEvent call or the next
106  geventGetSourceListener call.
107  - Note: All listeners must be processed for this event before anything else is processed.
108 */

164 /* @brief Register a callback for an event on a listener from an assigned source.
165  * @details The type of the event should be checked (pevent->type) and then pevent should be typecast to the
166  * actual event type if it needs to be processed.
167  *
168  * @params[in] pl The Listener
169  * @params[in] fn The function to call back
170  * @params[in] param A parameter to pass the callback function
171  *
172  * @note The GEvent buffer is valid only during the time of the callback. The callback MUST NOT save
173  * a pointer to the buffer for use outside the callback.
174  * @note An existing callback function is de-registered by passing a NULL for 'fn'. Any existing
175  * callback function is replaced. Any thread currently waiting using geventEventWait will be sent the exit event.
176  * @note Callbacks occur in a thread context but stack space must be kept to a minumum and
177  * the callback must process quickly as all other events are performed on a single thread.
178  * @note In the callback function you should never call ANY event functions using your own GListener handle
179  * as it WILL create a deadlock and lock the system up.
180  * @note Applications should not use this call - geventEventWait() is the preferred mechanism for an
181  * application. This call is provided for GUI objects that may not have their own thread.
182  */
183 void geventRegisterCallback(GListener *pl, GEventCallbackFn fn, void *param);
